I've added many new and exciting QoL features that the average modder
may appreciate. Note: Dawg Studio lacks support for some AVA features.
If you're using AVA, you'll have to use Autumn's editor for those.
If you have no idea what AVA is, don't worry.

The editor was originally created to be compatible with my own mod, but after
absolutely ZERO demand, I've decided to strip out all that yucky custom stuff
and release this as its own thing.

Dawg Studio has many brand new features to enhance your modding experience.
These features include:

- Dark Mode! Tired of BL burning your eyes out? No? That's too bad.
- New pictures (After opening the .jar file in an app like 7-Zip, you can find the images at \ca\noxid\lab\rsrc\.)
- Recently loaded file locations so you don't need to dig through the file browser when you swap projects.
- You can now change the sprite scale multiplier on the entity tab! (Useful for 2x)
- NPC sprites are now centered in the middle rather than positioned at the top left.
- You can now dock the script editor.
- Undo/Redo functionality in the script editor.
- New colors for commands, numbers, events, and more.
- Flags now have a separate color from numbers.
- Unrecognized commands are now highlighted in red. No more looking for that one typo that's crashing your game.
- The editor no longer reads the first digit of face portraits. (D-RS/CS+ QoL)
- Up to SIX operands are supported. (<SIXwwww:xxxx:yyyy:zzzz:qqqq:rrrr)
- String$ support for TSC commands. They are closed by the $ symbol.
- Event numbers and maps on the sidebar are clickable.
- Zoom in and out of the script window by holding control and moving the scroll wheel.
- Tabs can be dragged and reordered! Cool! (Standalone files will always appear on the far right)
- You can drag .tsc or .txt files into the script window tab bar. You can edit these standalone files alongside
the map script files, which may prove helpful when making large edits to any of the \data\ .tsc files.
- The npc.tbl editor has been massively improved, ridding of that unfinished category tree in favor of...
blank space.
- You can edit the entity names in the editor now.
- New sprite location input where you can plug in your sprite rects to have them show up in the editor.
- Copy/Paste functionality that copies hitbox, display, location, and flags to a string on your clipboard.
Press the paste button to paste the copied string back into the table and it'll replace the current attributes.
- New warning prompts when saving or closing the npc.tbl.
- A few bugfixes. Most notably, if you've loaded the stage file instead of the .exe, the editor
will no longer corrupt it when adding or deleting maps! Hopefully. Remember to always back up before you try this,
just in case.
